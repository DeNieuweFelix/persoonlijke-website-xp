ZybrovScript Official Docs!

Intro:

ZybrovScript was made VERY quickly and poorly, and sucks at a lot of things, You can only use 3 variables of each type, which adds some extra challenge
ZybrovScript is made 100% in JavaScript without any libraries or frameworks, and the source code can be found on: https://github.com/DeNieuweFelix/persoonlijke-website-xp (yes, it's fully open source!!!)

Variables:

ZybrovScript only has 3 variables of each type (int, string, boolean), these variables can be changed whenever you want.

Integers:

ZybrovScript can handle integers up to 9007199254740991 (the max safe JavaScript limit), I don't know what happens when you use number bigger than 9007199254740991, and to be honest, i'm too lazy to try.
The 3 available integers-variables are: int0, int1 and int2

<<1: setting the value of an integer:>>~

SetVar([any of the integer variables]) [value]~


<<2: The value of int1 is set to 100>>~

SetVar(int1) 100~

LogVar(int1)~

<<3: Output will be 100>>~


SetVar(int0) 123~

SetVar(int0) 67~

<<4: 123 becomes 67>>~

LogVar(int0)~

<<5: Output will be 67>>~


<<6: Integer variables can also be negative>>~

SetVar(int2) -100~

LogVar(int2)~

<<7: Output will be -100>>~

Strings:

Strings are used to store text (just like in JavaScript, or really any (modern) programming language), you can also store integers in [str] variables, but you won't be able to do math with them
Unfortunately, you're not able to use spaces without using other functions to replace certain symbols with spaces (STRrep())
The 3 available string-variables are: str0, str1 and str2

<<1: ZybrovScript strings can be set without using "">>~

SetVar(str0) hello-world~

LogVar(str0)~

<<2: Output will be "hello-world">>~


SetVar(str1) hello~

SetVar(str2) world~

LogVar(str1)~

LogVar(str2)~

<<3: Output will be "hello" and "world">>~

Booleans:

Booleans (bools) are used to store either true or false, you can't store integers or strings in a boolean variable. Bools are used to compare values
Booleans also cannot be combined using the CombVar() function!
The 3 available boolean-variables are: bool0, bool1 and bool 2

SetVar(bool1) true~

LogVar(bool1)~

<<1: Output will be true>>~


SetVar(bool0) 12345~

SetVar(bool1) hello-world~

SetVar(bool2) true~


LogVar(bool0)~

LogVar(bool1)~

LogVar(bool2)~

<<2: Nothing will be output, because wrong data types are used>>~

Functions:

Comments:

I'm gonna be honest with you, comments aren't really ""functions"", but I had to put them somewhere lol
Comments are used to add notes between code, and will be ignored.

SetVar(str0) meow~

<<SetVar(str0) meowmeow~>>~


LogVar(str0)~

<<1: The comment will be ignored, so the output is "meow" (and not "meowmeow")>>~


<<SetVar(int1) 100~>>~

<<SetVar(int2) 500~>>~

LogVar(int1)~

<<2: Output will be 0 (default integer value)>>~

SetVar()~

SetVar is used to assign a value to one of the variables, the correct syntax is: SetVar([variable]) [value]~ where variable is one of the variables, and value is the value you want to assign.

SetVar(int0) 500~

<<500 will be saved to int0>>~


SetVar(str1) hello-world~

<<"hello-world" will be saved to str1>>~


SetVar(bool2) true~

<<true will be saved to bool2>>~

ClearVar()~

ClearVar is used to reset (or clear) a variable. (0 for integers, "" for strings and false for booleans) the correct syntax is: ClearVar([variable])~

<<assigning values:>>~

SetVar(str1) meow~

SetVar(str2) woof~


<<str2 will be cleared>>~

ClearVar(str2)~


LogVar(str1)~

LogVar(str2)~

<<Output will be "meow" and "" (default string value)>>~


SetVar(int0) 150~

ClearVar(int0)~


LogVar(int0)~

<<Output will be 0 (default integer value)>>~


SetVar(bool0)~

ClearVar(bool0)~


LogVar(bool0)~

<<Output will be false (default boolean value)>>~

LogVar()~

The LogVar()~ function is used to output the value of variables to the console. The correct syntax is: LogVar([variable])~ LogVar()~ can be used on any data type

SetVar(int1) 500~

LogVar(int1)~

<<Output will be 500>>~


SetVar(str0) boooo~

LogVar(str0)~

<<Output will be "boooo">>~


SetVar(bool2) true~

LogVar(bool2)~

<<Output will be true>>~
